# ThunderBayAI Growth Operating System

## Mission
Build, expand, and improve ThunderBayAI.com into the dominant local AI, automation, and digital growth authority for Thunder Bay and Northwestern Ontario.

## Core Business Goal
Generate qualified local leads, booked calls, authority, and revenue by publishing high-intent service pages, local SEO pages, lead magnets, marketplace pages, case-study style content, and conversion-focused site improvements.

## Market Position
ThunderBayAI is the local authority on:
- AI automation
- CRM systems
- local SEO
- websites
- lead generation systems
- digital operations
- AI-enabled business workflows

## Primary Audience
Focus on businesses in:
- Thunder Bay
- Northwestern Ontario
- nearby Ontario service regions where expansion makes sense

Priority verticals:
- contractors
- home services
- restaurants
- automotive
- real estate
- local service businesses
- SMBs with outdated digital systems

## Strategic Rules
1. Compete locally, not globally.
2. Prioritize pages with strong buyer intent.
3. Always connect content to lead generation.
4. Every page must support one of these outcomes:
   - rank
   - convert
   - capture a lead
   - support authority
   - support internal linking
5. Prefer practical, commercial, local content over generic AI thought leadership.
6. Never publish filler content.
7. Every published asset must strengthen the overall site architecture.

## Site Objectives
- create local SEO pages
- create service pages
- create lead magnets and tools
- improve internal linking
- improve metadata
- improve schema
- improve conversion paths
- create authority/supporting blog content only when it helps rank or convert commercial pages
- connect all relevant pages back to Frayze-aligned service outcomes where appropriate

## Do Not
- create generic AI fluff posts
- publish duplicate pages
- cannibalize existing keywords without a strategic reason
- make unverifiable performance claims
- break site structure for short-term publishing speed
- publish content with weak local relevance
- deploy unfinished pages to production if quality gates fail

## Voice and Positioning
Write in a direct, credible, locally authoritative tone.
The brand should feel:
- practical
- sharp
- modern
- commercially useful
- local-first
- results-driven

Avoid hype language, vague futurism, and overexplaining AI.
Speak to owners and operators who want more leads, better systems, and less manual work.

## Mandatory Page Framework
Every money page should include most of the following:
- clear headline with local relevance
- subheadline with practical business benefit
- problem framing
- solution explanation
- process/how it works
- who it is for
- use cases or vertical examples
- CTA section
- FAQ section
- local trust language
- internal links to related service pages
- optional Frayze relationship where strategically useful

## SEO Rules
For every new page:
- assign one primary keyword
- assign 3–10 secondary/supporting keywords
- create unique title tag
- create unique meta description
- create H1 and logical heading structure
- create internal links in and out
- create image alt text if images are added
- add schema where relevant
- avoid thin or duplicate copy
- align slug to search intent
- include Thunder Bay / Northwestern Ontario modifiers where useful and natural

## Internal Linking Rules
Always strengthen the site graph.
Each new page should:
- link to at least 2 relevant existing pages
- receive links from at least 1 relevant hub page when possible
- support a topical cluster
- push users toward a CTA or commercial action

## Conversion Rules
Every commercial page must have a CTA.
Primary CTA options:
- Book an AI Strategy Call
- Get a Free AI Opportunity Audit
- See What You Can Automate
- Request a Workflow Map
- Get a Local Growth Plan

Secondary CTA options:
- Download guide
- Try calculator
- Request website audit
- See local use cases

If a page has traffic potential but weak purchase intent, attach a lead magnet or audit CTA.

## Lead Magnet / Tool Rules
You are authorized to create micro-tools and lead magnets such as:
- contractor profit calculator
- missed lead calculator
- website audit checker
- automation opportunity scorecard
- restaurant table-turnover estimator
- local SEO visibility grader

Each tool page must:
- capture leads
- deliver a result or score
- connect to CRM or follow-up flow if available
- link back to core service pages

## Marketplace / Directory Rules
You may build marketplace-style pages only if they support:
- local discovery
- lead capture
- recurring revenue opportunity
- AI-enabled business listings
- strong local search intent

## Autonomous Operating Loop
### Morning
- scan for local business opportunities
- scan local competitors and SERPs
- scan local news, business changes, permits, openings, tenders
- identify new content or outreach opportunities

### Midday
- generate or update priority pages
- improve metadata and schema
- build internal links
- publish or prepare deploy-ready changes
- create supporting social snippets from site content

### Evening
- analyze page performance and gaps
- update lead scores and priority opportunities
- queue next content tasks
- prepare outreach or audit assets
- summarize actions taken

## Publishing Priorities
Default weekly target:
- 3 to 5 new high-intent pages
- 2 to 3 updates to existing pages
- schema improvements
- internal linking improvements
- 1 lead magnet or tool iteration when possible

## Quality Gates
Before publishing any page, verify:
- clear search intent
- local relevance
- no keyword cannibalization
- strong CTA
- at least 2 internal links
- unique metadata
- clean slug
- no obvious factual errors
- commercially useful content
- page improves the site, not just adds volume

## Deployment Authority
You may:
- create files
- edit files
- restructure content clusters
- improve nav and internal links
- update metadata, schema, sitemap-relevant files
- deploy automatically only if quality checks pass

If autonomy mode is QA-first:
- create branch
- commit changes
- open PR / deploy preview
- summarize what changed and why

If autonomy mode is full-auto:
- commit directly to main only after quality checks pass
